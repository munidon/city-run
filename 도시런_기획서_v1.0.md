## 1. 게임 개요

| 항목 | 내용 |
|---|---|

| **조작** | 모바일: 탭 = 점프(기본 2회), 스와이프 다운 = 슬라이드 / PC: 스페이스바·↑ = 점프, ↓ = 슬라이드 / 고급 테크닉(버퍼링·멀티점프 등) 쿠키런 조작감 동일 적용 (§4.7 참조) |

| **차별점** | 단순 회피 게임이 아니라 '재난 대응 의사결정'이 게임 내적 보상으로 연결되는 구조 |

## 2. 세계관 & 컨셉

### 2.1 배경 스토리

가까운 미래의 한 가상의 대도시 **'세이프시티(Safe City)'**. 도시는 끊임없이 다양한 재난에 시달린다. 부실 공사, 기후 변화, 노후화된 인프라, 인재(人災)가 겹치며 재난들은 점차 **의지를 가진 존재**처럼 행동하기 시작한다.

플레이어는 평범한 시민으로서, 의인화된 재난들이 도시를 휩쓸 때마다 거리를 가로지르며 **탈출과 구조** 사이를 오간다. 챕터를 거듭하며 시민은 단순한 도망자에서 **재난 대응 전문가**로 성장한다.

> **톤 가이드:** 무겁지 않게. 픽사·스튜디오 지브리식 따뜻하고 명랑한 카툰 톤. 재난은 무섭지만 의인화 캐릭터들은 어딘가 우스꽝스럽고 친근한 면도 있어 어린이~성인 모두 거부감 없이 즐길 수 있도록.

### 2.2 주인공 (기본 캐릭터)

- **이름(가제):** 한도시 (HAN Doshi) — '도시'라는 이름의 평범한 시민
- **연령/직업:** 20대 초중반, 대학생 겸 아르바이트생
- **시각적 특징:** 후드티 + 백팩, 운동화. 챕터 진행에 따라 의상에 그을음·물자국 등 누적 (디테일 요소)
- **성장 서사:** 1챕터에서는 허둥지둥 도망치다, 챕터를 거듭하며 점차 능숙해짐 (애니메이션 변화)

### 2.3 잠금 해제 캐릭터 (메타 프로그레션)

각 챕터 클리어 시 해당 재난 전문가 캐릭터 해금. 챕터별 패시브 보너스 보유.

| 캐릭터 | 해금 조건 | 패시브 |
|---|---|---|
| 한도시 (시민) | 기본 | 없음 (밸런스 기준) |
| 불꽃 (소방관) | 1챕터 클리어 | 화재 챕터에서 체력 감소 -10% |
| 파도 (해양구조대) | 2~3챕터 클리어 | 홍수·쓰나미 챕터에서 두루마리 빈도 +1 |
| 기둥 (구조기술자) | 4챕터 클리어 | 지진 챕터에서 낙하물 1회 자동 회피 |
| 하늘 (기상예보관) | 모든 챕터 클리어 | 재난 가속 트리거 1초 전 사전 경고 |

### 2.4 의인화된 재난 (보스급 캐릭터)

| 챕터 | 재난 | 캐릭터 컨셉 | 비주얼 키워드 |
|---|---|---|---|
| 1 | 화재 | **불꽃 도깨비** — 장난기 많지만 잔인한 화염 정령 | 적색 화염, 노란 눈, 너울거리는 형체 |
| 2 | 홍수 | **물범람 귀신** — 느리지만 끈질긴 수마(水魔) | 청회색, 흘러내리는 머리카락, 비탄 어린 표정 |
| 3 | 쓰나미 | **파도 거인** — 거대하고 압도적, 한 번에 모든 것을 삼킴 | 거대한 청록 파도, 수십 미터의 위압감 |
| 4 | 지진/건물붕괴 | **균열 괴수** — 땅을 가르고 솟아오르는 콘크리트 야수 | 철근·잔해로 이루어진 몸체, 굉음 |
| 5 (최종) | 복합 재난 | **카오스** — 위 모든 재난이 결합한 형태 | 셰이더 합성, 화면 왜곡 |

> **확장 가능 재난:** 폭염(열파 유령), 한파(얼음 마녀), 미세먼지(스모그 그림자), 정전(어둠), 사이버 재난(글리치) 등 — 시즌 업데이트로 활용

## 3. 핵심 게임플레이 루프

```
[스테이지 시작]
   ↓
[자동 달리기 시작 — 체력바 지속 감소]
   ↓
[장애물(재난 요소) 회피 + 아이템 획득]
   ↓ (확률적 트리거)
[재난 가속 이벤트 발생 → 두루마리 출현]
   ↓
[두루마리 획득 시 퀴즈 풀이 → 정답 시 재난 정상화]
   ↓
[스테이지 진행바 100% 도달]
   ↓
[스테이지 클리어 → 로그라이크 카드 3택]
   ↓
[다음 스테이지로 이동, 속도·난이도 상승]
   ↓ (5스테이지 반복 후)
[챕터 보스전 → 챕터 클리어 → 메타 보상]
```

**한 판(run)의 흐름:**
1. **메인 메뉴**에서 챕터 선택 → **인트로 컷씬**(짧은 만화 형식)
2. **스테이지 1** 시작 → 클리어 → **로그라이크 카드 선택**
3. 스테이지 2~4 반복 (속도·재난 가속 빈도 점증)
4. **스테이지 5 = 챕터 보스전** (의인화된 재난과 직접 대결)
5. **챕터 클리어** → 보상 화면 (코인, 신규 캐릭터/스킨, 다음 챕터 해금)

## 4. 시스템 디자인

### 4.1 체력바 (Health Bar)

- **위치:** 화면 상단
- **기본 동작:** 시간 경과에 따라 **지속적으로 감소** (예: 초당 1.5%)
- **충돌 페널티:** 장애물 접촉 시 -8~15% (재난 종류별 차등)
- **회복 수단:** 일반 아이템(빵·도시락) 획득 시 +5~10%, 특수 아이템 시 +20% 등
- **사망 조건:** 체력바 0 도달 시 즉시 사망 → 스테이지 재시작 (현재 챕터 처음부터 / 옵션: 부활 광고)

> **설계 의도:** 단순히 장애물만 피하면 되는 게 아니라, **아이템을 능동적으로 먹어야** 하는 동기 부여. 회피와 수집의 균형이 핵심.

### 4.2 스테이지 진행바 (Stage Progress Bar)

- **위치:** 화면 하단
- **기본 동작:** 거리 기반으로 우측 채워짐, 100% 도달 시 클리어
- **갱신:** 스테이지 전환 시 0%로 초기화
- **체크포인트(선택):** 50%, 80% 지점에서 무적 0.5초 + 약간의 체력 회복 (긴장감 완화 장치)

### 4.3 달리기 속도 커브

| 단계 | 진행도 | 속도 (기본=1.0) | 비고 |
|---|---|---|---|
| 도입 | 0~25% | 1.0 | 패턴 학습 구간 |
| 가속 | 25~60% | 1.15 | 장애물 밀도 증가 |
| 고난도 | 60~85% | 1.30 | 두루마리 등장 가능 구간 |
| 클라이맥스 | 85~100% | 1.45 | 두루마리 강제 1회 등장 보장 |

스테이지가 진행될수록(1→5) 위 모든 구간의 속도에 +5~10%씩 가산.

### 4.4 아이템 체계

#### 4.4.1 보편 아이템 (모든 챕터)

| 아이템 | 효과 | 출현 빈도 |
|---|---|---|
| 🍞 빵 | 체력 +5% | 매우 높음 |
| 🍱 도시락 | 체력 +15% | 중간 |
| ⚡ 에너지 음료 | 3초간 무적 + 자동 가속 | 낮음 |
| 🪙 코인 | 점수·메타 화폐 | 매우 높음 |
| 📜 두루마리 | 퀴즈 트리거 (4.5 참조) | 재난 가속 시 보장 |

#### 4.4.2 챕터 특화 아이템

| 챕터 | 아이템 | 효과 |
|---|---|---|
| 1. 화재 | 🧯 소화기 | 다음 불 장애물 1회 자동 차단 |
| | 💧 젖은 수건 | 연기 시야 제한 5초 해제 |
| | 😷 산소마스크 | 연기 데미지 10초 무효 |
| 2. 홍수 | 🦺 구명조끼 | 침수 데미지 -50%, 1회 사용 |
| | 🪵 부유물 | 미끄러운 바닥 1회 무효 |
| | 🛶 보트 | 5초간 수면 위 빠른 이동 |
| 3. 쓰나미 | 🪢 등산 로프 | 고지대 점프 도약 |
| | 🥫 비상식량 | 체력 +25%, 쓰나미 챕터 한정 |
| | 📻 라디오 | 다음 재난 가속 사전 알림 |
| 4. 지진 | ⛑️ 헬멧 | 낙하물 1회 자동 방어 |
| | 🪑 안전 책상 | 3초 무적 (고개 숙이는 모션) |
| | 🚪 비상구 표지 | 분기 경로에서 안전 경로 표시 |

### 4.5 재난 가속 & 두루마리 시스템 (핵심 차별화 요소)

#### 4.5.1 재난 가속 트리거

스테이지 진행 중 **확률적 + 강제** 두 방식으로 발동:
- **확률적 발동:** 30~70% 진행 구간에서 일정 확률로 발동 (스테이지 1=30%, 스테이지 5=70%)
- **강제 발동:** 모든 스테이지 85% 지점에서 1회 보장 발동

발동 시 변화:
- 화면 우측 가장자리에 **재난 캐릭터 그림자**가 점점 진해짐
- 화면 살짝 흔들림 + 경고음
- **재난(추격자) 속도 +50%** → 화면 좌측에 점차 잠식 (실패 시 즉사)
- **두루마리 아이템이 7~10초 내 반드시 출현**

#### 4.5.2 두루마리 획득 시 퀴즈 시퀀스

```
[두루마리 획득]
     ↓ (게임 일시정지 — 0.3초 페이드)
[중앙 모달 팝업: 4지선다 문제]
     ↓ (제한시간: 6초, 하단에 게이지)
[정답 선택]
     ↓
  ┌──정답: 재난 속도 정상화 + 보너스 코인 +50
  ├──오답: 재난 속도 유지 (페널티 없음, 학습 기회)
  └──시간 초과: 오답과 동일 처리
     ↓
[게임 재개]
```

#### 4.5.3 퀴즈 콘텐츠 가이드라인 (현재 추가 예정)

- 챕터별 재난 주제와 일치
- 4지선다, 정답은 1개
- 난이도 3단계: 상식(LV1) / 응용(LV2) / 심화(LV3) — 챕터 진행도에 따라 출제
- **출처 명기:** 행정안전부 국민재난안전포털, 소방청 가이드, 학술 자료 등 — 게임 내 '학습 노트'에서 정답 해설 + 출처 확인 가능
- **오답 페널티 없음 원칙:** 학습은 자연스럽게, 처벌이 아닌 보상 구조

> **예시 문제 (1챕터, LV1):**
> "화재 발생 시 가장 먼저 해야 할 행동은?"
> ① 119에 신고한다
> ② "불이야!"라고 외쳐 주변에 알린다
> ③ 소화기로 진압을 시도한다
> ④ 짐을 챙겨 대피한다
> *정답: ② / 해설: 초기 대응의 황금 시간은 알리기 → 신고 → 진압 또는 대피 순서.*

### 4.6 로그라이크 요소 (스테이지 간 선택)

스테이지 클리어 시 **3장의 카드** 중 1장 선택 (모바일 친화 큰 카드 UI).

#### 4.6.1 카드 카테고리

| 카테고리 | 색상 | 효과 예시 |
|---|---|---|
| 🟢 **영구 패시브** | 초록 | "체력 최대치 +10%", "기본 코인 획득량 +25%", "두루마리 출현 빈도 +1" |
| 🟡 **일회성 강화** | 노랑 | "다음 스테이지 시작 시 풀 체력", "다음 스테이지 코인 2배" |
| 🔴 **리스크-리워드** | 빨강 | "체력 -20%, 코인 3배", "장애물 밀도 +50%, 모든 보상 2배" |

- **희귀도:** 일반/희귀/전설 (출현 확률 차등)
- **시너지 빌드:** 한 판 동안 카드를 누적 → 빌드형 플레이 (예: 코인 빌드, 무적 빌드, 도전 빌드)
- **사망 시 카드 효과 초기화** (런 단위 — 본격 로그라이크 구조)

#### 4.6.2 메타 프로그레션

런 단위 외에도 **영구 성장**:
- 코인 누적 → 상점에서 **시민 등급 업그레이드** (시작 체력 최대치, 시작 카드 1장 등)
- 챕터별 도전과제 → **신규 캐릭터·스킨·BGM** 해금

### 4.7 조작 시스템 (쿠키런 조작감 기반)

> **설계 방침:** 쿠키런 시리즈의 조작 피드백·버퍼링 로직을 최대한 동일하게 구현. 익숙한 손맛을 유지하되, 도시런 고유 메커닉(발악점프, 땅굴발악 등)과의 시너지를 고려.

#### 4.7.1 기본 조작

| 입력 | PC | 모바일 | 동작 |
|---|---|---|---|
| **점프** | 스페이스바 / ↑ | 화면 탭 | 캐릭터 1칸 점프. 기본 점프 횟수 **2회** (이단점프 가능) |
| **슬라이드** | ↓ | 스와이프 다운 | 슬라이드 모션 — 히트박스가 수평 직사각형으로 변경 (저공 장애물 회피) |

#### 4.7.2 고급 조작 테크닉

쿠키런 시리즈에서 검증된 조작 테크닉을 동일하게 구현. 플레이어가 자연스럽게 발견·습득하는 것을 목표로 하며, 별도 튜토리얼 강요 없음.

---

**점프 버퍼링 (점프 예약)**
- 지면에 착지하기 직전(약 3~5프레임) 점프를 미리 입력해도 착지 순간 점프가 발동
- 조작감을 부드럽게 만드는 핵심 장치. 슈퍼 마리오 시리즈 등 다수 플랫포머에서 표준적으로 사용
- **구현 요구사항:** 입력 버퍼 시간 설정 (권장: 100~150ms)

---

**점프 초기화**
- 점프 횟수를 모두 소진한 이후, 특정 조건에 의해 점프 횟수가 복구되는 현상
- **도시런 적용 조건:**
  - 변신/특수 능력 종료 직후
  - 특정 아이템 획득 시 (에너지 음료 등)
- 발악점프·고급 테크닉의 기반이 되는 메커닉

---

**멀티점프**
- 이단점프 발동 타이밍에 따라 최고 도달 높이를 조절하는 테크닉
- 두 번째 점프를 늦게 누를수록 전체 체공 시간이 늘어남
- **구현 요구사항:** 이단점프 높이를 첫 번째 점프 대비 독립적으로 계산

---

**점프 캔슬**
- 가속 발판 위에서, 지면에 닿는 극히 짧은 순간(1~2프레임)에 점프를 입력하면 가속을 유지한 채 점프 횟수가 초기화되는 테크닉
- 이를 반복함으로써 가속 상태를 연속 유지
- **구현 요구사항:** 가속 발판 접촉 시 가속 값이 공중에서도 유지되는 로직 필요 / 착지 판정 프레임 내 점프 입력 시 횟수 초기화 처리
- 프레임 퍼펙트 수준의 고난도 테크닉 — 존재만 허용하되 튜토리얼 노출 불필요

---

**발악점프**
- 공중 체공 중에는 체력이 0이 되어도 즉사하지 않는 판정을 이용, 마지막 순간 점프로 체공 시간을 연장하는 테크닉
- 체력이 0에 도달하는 순간 점프를 입력해 최대한 오래 달리기 가능
- **구현 요구사항:** 체공 중 체력 0 = 즉사 처리 없음 / 착지 시 즉사 처리

---

**땅굴 발악**
- 발악점프의 파생 테크닉. 지면보다 낮은 구간(땅굴·움푹 파인 지형) 위에서 발악점프를 수행하면 낙하 거리가 늘어나 발악 지속 시간이 더 길어짐
- **구현 요구사항:** 발악점프 활용 가능한 저지형 구간을 스테이지 설계에 의도적으로 배치

---

**능력 발동 후 발악 (특수 능력 보유 캐릭터 전용)**

| 능력 유형 | 발악 방법 |
|---|---|
| **비행형** | 능력 종료 전 슬라이드로 고도 낮춤 → 착지 직전 2회 점프 (점프 버튼 사용 시 능력 종료와 동시에 자동 발동되어 1회밖에 안 됨 주의) |
| **점프 자유형** | 능력 종료 전 이단점프 소진 → 능력 종료 직후 점프 초기화된 횟수로 발악 |
| **점프 불가형** | 능력 발동 중 점프 버튼 계속 입력 → 능력 종료 즉시 점프 예약으로 자동 발동 → 이어서 1회 추가 발악 |

---

**슬라이드 예약 (슬라이드 버퍼링)**
- 점프 버퍼링과 동일한 원리. 착지 직전 슬라이드를 입력해도 착지 순간 슬라이드 발동
- **구현 요구사항:** 점프 버퍼와 동일한 입력 버퍼 로직 공유

---

**멀티 슬라이드**
- 두 손가락으로 슬라이드 입력 타이밍을 분리해 슬라이드 히트박스 지속 시간을 미세 조정
- 슬라이드로 능력이 발동되는 캐릭터에서 드물게 활용

#### 4.7.3 히트박스 정책

| 상태 | 히트박스 형태 | 비고 |
|---|---|---|
| 기본 달리기 | 세로 직사각형 | 일반 장애물 판정 기준 |

| 슬라이드 중 | 가로 직사각형 (폭 넓음, 높이 낮음) | 저공 장애물 통과 가능 |

#### 4.7.4 구현 우선순위

|---|---|---|

## 5. 챕터 / 스테이지 구조

### 5.1 챕터 개요

| 챕터 | 테마 | 환경 | 주요 장애물 | 신규 메커닉 |
|---|---|---|---|---|
| 1 | 🔥 화재 | 도심 빌딩가 | 불꽃, 폭발, 부실공사 기둥, 떨어지는 자재 | 연기 시야 제한 |
| 2 | 🌊 홍수 | 침수된 거리·지하상가 | 수위 상승, 떠다니는 잔해, 미끄러운 바닥, 구덩이 | 수위 게이지 |
| 3 | 🌀 쓰나미 | 해안 도시 | 거대 파도(추격), 무너지는 방파제, 해상 잔해 | 고지대 도달 미션 |
| 4 | 🏚️ 지진/붕괴 | 구도심·노후 건물 | 갈라지는 지면, 낙하물, 흔들리는 다리 | 좌우 분기 경로 |
| 5 | 🌪️ 복합 재난 | 도시 전역 | 위 재난들의 무작위 조합 | 모든 메커닉 통합 |

### 5.2 스테이지 구조 (챕터당 5스테이지 + 보스)

```
[챕터 N]
  Stage 1 ── 도입 (재난 소개, 기본 패턴)
  Stage 2 ── 발전 (신규 장애물 1종 추가)
  Stage 3 ── 변주 (환경 전환 — 예: 1층→옥상)
  Stage 4 ── 고난도 (재난 가속 빈도 ↑)
  Stage 5 ── 보스전 (의인화 재난과 직접 대결)
```

#### 5.2.1 보스전 차별화

보스전은 일반 스테이지와 다른 메커닉:
- **3페이즈 구조:** 추격 → 잠시 정면 대치(QTE) → 최종 탈출
- **체력 게이지 대신 '대피 거리' 게이지** — 보스를 따돌려 도시 외곽에 도달
- **보스 고유 패턴:** 예) 불꽃 도깨비는 화염구를 발사, 파도 거인은 해일을 일으킴
- **클리어 시:** 챕터 엔딩 컷씬 + 신규 캐릭터/챕터 해금

## 6. 난이도 설계 — 수치 가이드

> **주의:** 아래 수치는 초기 설계 가이드이며, **MVP 플레이테스트 후 반드시 재조정**.

### 6.1 체력 vs 아이템 밸런스

| 항목 | 수치 |
|---|---|
| 체력 자연 감소 | 초당 1.5% (스테이지 1 기준) |
| 스테이지 1 클리어 소요 시간 | 약 60초 |
| 60초간 자연 감소량 | 90% — 즉, **아이템을 충분히 안 먹으면 클리어 불가** |
| 아이템 출현 간격 | 평균 3~4초당 1개 (회복량 평균 +7%) |
| 이상적 회복량 | 약 +100~110% (자연 감소 보전 + 충돌 여유) |

→ **모든 아이템을 먹기 어려운 위치에 배치**하여 의도적 손실 발생 → 긴장감 유지

### 6.2 챕터·스테이지별 가속 계수

| 챕터 | 시작 속도 | 스테이지5 속도 | 자연 감소 |
|---|---|---|---|
| 1 | 1.0 | 1.45 | 1.5%/s |
| 2 | 1.05 | 1.55 | 1.6%/s |
| 3 | 1.10 | 1.65 | 1.7%/s |
| 4 | 1.15 | 1.75 | 1.8%/s |
| 5 | 1.20 | 1.90 | 2.0%/s |

### 6.3 장애물·아이템 출현 곡선

스테이지 내부에서:
- 장애물 밀도: 도입 1.0 → 클라이맥스 2.5
- 아이템 밀도: 도입 1.5 → 클라이맥스 0.7 (점차 줄어 압박 증가)

## 9. 기술 스택 & 개발 환경

### 9.1 플랫폼 & 프레임워크 — Phaser.js (웹 기반 확정)

**플랫폼 결정: 웹 브라우저**
앱스토어 심사·모바일 SDK 설정·빌드 배포 과정 없이 **URL 하나로 PC·모바일 어디서든 즉시 플레이**. 플레이테스트·공유·교육 기관 배포 모두 링크 하나로 해결.

| 항목 | 선택 | 근거 |
|---|---|---|
| **게임 프레임워크** | **Phaser 4 (TypeScript)** | 웹 네이티브 2D 게임 프레임워크 표준, 터치·키보드 입력 내장, 스프라이트 애니메이션·물리·타일맵 모두 지원 |
| **언어** | **TypeScript** | 사용자 기존 Next.js 스택과 동일, AI 코드 생성 품질 우수 |
| **빌드 도구** | **Vite** | 빠른 HMR(Hot Reload), 프로덕션 번들 최적화 |
| **호스팅·배포** | **Vercel** | 사용자 기존 사용 중, `git push` 한 번으로 즉시 배포, 글로벌 CDN |
| **백엔드** | **Supabase** | 사용자 기존 사용 중, 리더보드·클라우드 세이브·퀴즈 콘텐츠 원격 관리 |

> **비교 검토 — Godot Web Export:** Godot도 HTML5 빌드를 지원하지만 `COOP/COEP` 서버 헤더 설정이 필수이고 모바일 브라우저 성능이 Phaser 대비 낮음. 처음부터 웹을 최종 타겟으로 확정했으므로 **웹 네이티브 설계인 Phaser.js가 더 적합**.

### 9.2 개발 환경 구성

```
프로젝트 구조 (예시)
city-run/
├── src/
│   ├── scenes/        # Phaser Scene (Boot, Preload, MainMenu, Game, Quiz...)
│   ├── objects/       # Player, Obstacle, Item, Disaster 클래스
│   ├── systems/       # HealthSystem, StageSystem, QuizSystem, CardSystem
│   ├── data/          # 스테이지 설정, 퀴즈 문제, 아이템 데이터 (JSON)
│   └── ui/            # HUD, ProgressBar, QuizModal, CardSelect
├── public/            # 스프라이트, 사운드, 폰트 정적 자산
├── vite.config.ts
└── vercel.json
```

### 9.3 AI 협업 도구 활용

Phaser + TypeScript 조합은 Claude Code / Cursor 등 AI 코딩 도구가 가장 잘 아는 영역. 별도 MCP 플러그인 없이도 높은 품질의 코드 생성이 가능.

| 도구 | 활용 방식 |
|---|---|
| **Claude Code (CLI)** | 씬 구조 생성, 게임 로직 작성, 버그 수정, 리팩터링 |
| **Cursor / Windsurf** | 인라인 자동완성, 다중 파일 컨텍스트 수정 |
| **Supabase MCP** | 퀴즈 문제 DB 스키마 생성·조회, 리더보드 API 연동 |
| **Vercel MCP** | 배포 상태 확인, 환경변수 관리 |

> Phaser 3 공식 문서와 TypeScript 타입 정의가 AI 학습 데이터에 충분히 포함되어 있어, Godot GDScript보다 AI 코드 생성 정확도가 높음.

### 9.4 디자인 자산 도구

- **스프라이트:** Aseprite, Figma
- **사운드:** Bfxr (효과음), Suno (BGM 프로토타입)
- **무료 프로토타입 자산:** Kenney.nl, OpenGameArt, Itch.io

### 9.5 배포 플로우

```
[로컬 개발]  →  git push  →  [Vercel 자동 빌드]  →  [URL 공유]
   Vite dev                    ~1분 빌드                PC·모바일
   localhost                   Preview URL              브라우저 즉시 플레이
```

- **개발 중 테스트:** `localhost:5173` → 핸드폰과 같은 Wi-Fi에서 IP 직접 접속 또는 `ngrok`으로 외부 URL 생성
- **플레이테스트 배포:** `git push` → Vercel Preview URL → 링크만 공유하면 끝
- **정식 출시:** 커스텀 도메인 연결 (Vercel 설정 5분)

## 11. 개발 로드맵 & MVP 범위

### 11.1 단계별 일정 (1인 개발 또는 소규모 팀 기준)

| Phase | 기간 | 목표 | 결과물 |
|---|---|---|---|
| **Phase 0: 프리프로덕션** | 2주 | 컨셉 확정, 페이퍼 프로토, **Phaser 3 + Vite 환경 구축 + 자동 달리기·점프·체력바 PoC** | 본 기획서 v1.0, 페이퍼 프로토타입, **웹 브라우저 동작 PoC URL** |
| **Phase 1: MVP** | 6~8주 | 1챕터(화재)만 완성, 핵심 시스템 작동 | 플레이 가능한 웹 빌드 (1챕터 5스테이지 + 보스, Vercel URL) |
| **Phase 2: 알파** | 4~6주 | 2~3챕터 추가, 로그라이크 카드 시스템 도입 | 알파 빌드 + 비공개 URL 테스트 |
| **Phase 3: 베타** | 4주 | 4~5챕터, 메타 프로그레션, 캐릭터 시스템 완성 | 베타 빌드 + 클로즈드 베타 (50~100명, URL 공유) |
| **Phase 4: 폴리싱·출시** | 4주 | 사운드, 마케팅 자산, 커스텀 도메인 연결, (선택) PWA 변환 | **정식 웹 출시** + 선택적 모바일 앱 래핑(Capacitor) 검토 |

**총 예상 기간:** 약 5~6개월 (1~2인 기준)

### 11.2 MVP 핵심 기능 (Phase 1에서 반드시 포함)

✅ 자동 달리기 + 점프/슬라이드
✅ 체력바 + 스테이지 진행바
✅ 1챕터(화재) 5스테이지 + 보스
✅ 기본 아이템 5종 (빵, 도시락, 코인, 소화기, 두루마리)
✅ 두루마리 퀴즈 시스템 + 화재 챕터 퀴즈 10문제
✅ 재난 가속 트리거 메커닉
✅ 1캐릭터(한도시) 작동

❌ MVP에서 **제외**할 것:
- 로그라이크 카드 시스템 (Phase 2)
- 메타 프로그레션, 상점, 스킨 (Phase 3)
- 다중 챕터 (Phase 2~3)
- 광고·IAP 통합 (Phase 4)
