## 1. 게임 개요

| 항목 | 내용 |
|---|---|
| **장르** | 2D 사이드뷰 러닝 액션 + 라이트 로그라이크 + 퀴즈 요소 |
| **플랫폼** | 웹 브라우저 (PC·모바일 모두 지원) |
| **조작** | ✅ PC: Space/↑/W = 점프(2단), ↓/S = 슬라이드(홀드) / ✅ 모바일: 좌측 원형 JUMP 버튼, 우측 원형 SLIDE 버튼(홀드) / ⏳ 고급 테크닉(버퍼링·발악점프 등 §4.7) — Phase 1~3 분배 |
| **타겟층** | 1차: 10~20대 캐주얼 게이머 / 2차: 도시안전 교육 관심 학교·기관 |
| **세션 길이** | 1스테이지 60초 (`STAGE_DURATION_SEC`) / 1챕터 ≈ 6~8분 |
| **핵심 컨셉** | "쿠키런처럼 달리되, 쫓아오는 것은 의인화된 도시 재난이며, 살아남기 위해선 재난 상식까지 알아야 한다." |
| **차별점** | 단순 회피 게임이 아니라 '재난 대응 의사결정'이 게임 내적 보상으로 연결되는 구조 |

## 2. 세계관 & 컨셉 (v1.0 유지)

### 2.1 배경 스토리

가까운 미래의 가상 대도시 **'세이프시티'**. 부실 공사·기후 변화·노후 인프라가 겹치며 재난들이 의지를 가진 존재처럼 행동하기 시작한다. 플레이어는 평범한 시민이지만, 챕터를 거듭하며 단순 도망자에서 **재난 대응 전문가**로 성장한다.

> **톤 가이드:** 무겁지 않게. 픽사·지브리식 따뜻하고 명랑한 카툰 톤.

### 2.2 주인공

- **이름(가제):** 한도시 (HAN Doshi)
- ✅ 현재 사용 자산: `public/assets/player/player_boy.{png,json}` (러닝 8프레임, hit 2프레임)
- ✅ 점프 자산: `player_boy_jump.{png,json}` (6프레임 + hit 2프레임)
- ✅ 슬라이드 자산: `player_slide_sprites.{png,json}` (3프레임 + hit 2프레임)
- 🔄 `player_cat.{png,json}` 보유 — 아직 게임에서 미사용 (대체 캐릭터 후보)

### 2.3 잠금 해제 캐릭터 (메타 프로그레션) ⏳ Phase 3

| 캐릭터 | 해금 조건 | 패시브 | 상태 |

| 불꽃 (소방관) | 1챕터 클리어 | 화재 챕터에서 체력 감소 -10% | ⏳ |

## 3. 핵심 게임플레이 루프

```
[BootScene] → [MainMenuScene] → [GameScene] → (스테이지 클리어) → [CardSelectScene] → [GameScene]
              └── [SettingsScene] → [AssetPreviewScene]
```

상세 루프 (`src/scenes/GameScene.ts:131-163`):

```
[스테이지 시작]
  ↓
[자동 달리기 — 체력 1.5%/s 자연 감소]            ← HealthSystem.tick
  ↓
[장애물(박스/기둥) 회피 + 아이템(김밥·도시락·코인) 획득]   ← ObstacleSpawner / ItemSpawner
  ↓ (확률 30% @ 진행 10% / 강제 75%)
[재난 가속 발동 → 좌측 ChaseShadow 잠식 시작]    ← DisasterSystem.trigger
  ↓ (7~10초 내)
[두루마리 출현 → 픽업 시 QuizModal]              ← Scroll → QuizModal
  ↓
[정답 → +50 코인 + disaster.resolve / 오답·시간초과 → 페널티 없음]
  ↓
[60초 도달 → 스테이지 클리어]                    ← StageSystem.complete
  ↓
[CardSelectScene — 3장 중 1장 선택, RunState 누적]
  ↓
[다음 스테이지 (stageIndex+1, 속도 +7%)]         ← currentSpeed()
```

> ⏳ Phase 1 잔여: 챕터 선택 화면 / 인트로 컷씬 / 스테이지 5 보스전 / 사망 시 부활 광고 옵션

## 4. 시스템 디자인

### 4.1 체력바 ✅

- **위치:** 화면 상단 (`src/ui/HUD.ts:107`)
- **자연 감소:** 1.5%/s (HealthSystem.drainPerSecond)
- **충돌 페널티:** 박스 -10%, 기둥 -15%
- **회복:** 김밥 +2%, 도시락 +5% (run.healMul · run.pendingRewardMul 적용)
- **사망:** 체력 0 도달 또는 ChaseShadow에 잡힘 → 즉시 사망 → 챕터 처음부터 (RunState.reset())
- ✅ i-frame 700ms (충돌 직후) + 카메라 셰이크 + 빨간 화면 플래시
- ⏳ 부활 광고 옵션 (Phase 4)

### 4.2 스테이지 진행바 ✅

- **위치:** 화면 하단 (`src/ui/HUD.ts:120`)
- **로직:** `elapsedMs / (60s × 1000)` (StageSystem)
- **체크포인트:** 50%, 80% 도달 시 **+10 HP, 500ms 무적** (`GameScene.handleCheckpoint`)
  - v1.0 "약간의 회복"보다 풍부한 보상으로 구현됨

### 4.3 달리기 속도 커브 ✅

| 단계 | 진행도 | speedMultiplier | 비고 |
|---|---|---|---|
| 도입 | 0~25% | 1.00 | 패턴 학습 |
| 가속 | 25~60% | 1.15 | 장애물 밀도 ↑ |
| 고난도 | 60~85% | 1.30 | 두루마리 등장 가능 구간 |
| 클라이맥스 | 85~100% | 1.45 | 재난 가속 강제 발동(75%) 후 |

스테이지별 가산: `1 + (stageIndex - 1) × 0.07` (예: Stage 5 → +28%)

### 4.4 아이템 체계

#### 4.4.1 보편 아이템 (현재)

| 아이템 | 효과 | 출현 빈도 | 상태 |
|---|---|---|---|
| 🍙 김밥 (gimbap) | 체력 +2% | 매우 높음 (가중치 6) | ✅ |
| 🍱 도시락 (bento) | 체력 +5% | 중간 (가중치 2) | ✅ |
| 🪙 코인 (coin) | +1 코인 | 매우 높음 (가중치 5) | ✅ |
| 📜 두루마리 (scroll) | 퀴즈 트리거 | 재난 가속 시 7~10초 내 1회 보장 | ✅ |
| ⚡ 에너지 음료 | 3초 무적 + 가속 | — | ⏳ Phase 2 |

> **균형 메모:** 현재 회복량(+2/+5)은 v1.0(+5/+15)보다 보수적. 자연 감소 90%/60s를 아이템으로 보전하려면 평균 8개/스테이지 픽업 필요 — 의도된 긴장감.

#### 4.4.2 챕터 특화 아이템 ⏳ Phase 2~3

v1.0에 명세된 항목들. 자산은 일부 보유:
- 🧯 소화기, 💧 젖은 수건, 😷 산소마스크 (1챕터)
- 🦺 구명조끼, 🪵 부유물, 🛶 보트 (2챕터)
- 🪢 등산 로프, 🥫 비상식량, 📻 라디오 (3챕터)
- ⛑️ 헬멧, 🪑 안전 책상, 🚪 비상구 표지 (4챕터)

> 자산 폴더에 `flame.svg`, `falling-debris.svg`, `low-bar.svg` 가 있으나 현재 사용되지 않음 → Phase 2 화재 챕터 보강 시 재활용 검토.

### 4.5 재난 가속 & 두루마리 시스템 ✅ (수치 갱신)

#### 4.5.1 재난 가속 트리거 (`src/systems/DisasterSystem.ts:16-27`)

- **확률 발동:** 진행 10% 시점에 1회 주사위, **30% 확률로** 발동
- **강제 발동:** 진행 **75%** 도달 시 무조건 발동
- **발동 효과:**
  - 화면 좌측 ChaseShadow(그라디언트 + Fire Spirit 애니메이션 + 22 입자) 등장 (`src/objects/ChaseShadow.ts`)
  - 카메라 셰이크 400ms (강도 0.005)
  - HUD에 "⚠ 재난 출현 — 두루마리를 찾아라!" 표시
  - 게임 속도 +30% (DisasterSystem.speedBonus)
  - 두루마리 7~10초 내 1회 강제 출현
  - ChaseShadow가 17초 동안 좌측 -240px → +448px 까지 잠식 (플레이어 좌측 가장자리에 닿으면 즉사)
- **해소:** 두루마리 퀴즈 정답 시 ChaseShadow가 320 px/s로 후퇴

> ⏳ v1.0 "스테이지별 가변 확률(30%→70%)"은 현재 단일 30%로 고정. Phase 2에서 챕터·스테이지별 컨피그화 예정.

#### 4.5.2 두루마리 획득 → 퀴즈 시퀀스 ✅

시간 초과: 오답과 동일 처리
   ↓
[해설 패널 — 정답 / 해설 / 출처 노출 → 탭으로 닫기]
   ↓

## 6. 난이도 설계 — 수치 가이드 (v2.0 갱신)

> v1.0의 수치 가이드를 실제 코드 기준으로 갱신. 변경 표는 부록 B.

### 6.1 체력 vs 아이템 밸런스

| 항목 | 수치 |
|---|---|
| 체력 자연 감소 | 1.5%/s (HealthSystem) |
| 스테이지 1 클리어 소요 | 60s |
| 60초간 자연 감소량 | 90% |
| 김밥 회복 | +2% (가중치 6) |
| 도시락 회복 | +5% (가중치 2) |
| 코인 출현 | 가중치 5 |
| 충돌 페널티 | 박스 -10% / 기둥 -15% |
| 체크포인트 회복 | +10 HP @ 50%, 80% |

→ 현재 보수적인 회복량 → 풀체력 클리어 어려움. **아이템 미스 + 충돌 1~2회 = 사망권** 의도된 압박.

### 6.2 챕터·스테이지별 가속 계수

| 챕터 | 시작 속도 | Stage5 속도 | 자연 감소 | 상태 |
|---|---|---|---|---|
| 1 | 1.0 | 1.45 + (5-1)×7% = **1.73** | 1.5%/s | ✅ |
| 2 | 1.05 | 1.55 | 1.6%/s | ⏳ |
| 3 | 1.10 | 1.65 | 1.7%/s | ⏳ |
| 4 | 1.15 | 1.75 | 1.8%/s | ⏳ |
| 5 | 1.20 | 1.90 | 2.0%/s | ⏳ |

> ⏳ Phase 2: HealthSystem.drainPerSecond를 챕터별 컨피그로 외부화 필요.

### 6.3 장애물·아이템 출현 곡선 (`src/systems/ObstacleSpawner.ts:16-21`)

| 진행도 | 장애물 인터벌 | 가중치 (box : pillar) |
|---|---|---|
| 0~25% | 1700~2400ms | 6 : 2 |
| 25~60% | 1300~1900ms | 5 : 3 |
| 60~85% | 1050~1500ms | 4 : 4 |
| 85~100% | 900~1250ms | 4 : 5 |

아이템: 1400~2200ms, 가중치 김밥4 : 도시락2 : 코인5

- 챕터 선택 화면 (메인메뉴 → 챕터 1만 활성)
- 인트로 컷씬 1장 (정적 일러스트 + 텍스트)
- localStorage 영구 저장 (totalCoins, 해금 진행)
- 인터랙티브 튜토리얼 (스테이지 1 첫 30초)

### 11.2 Phase 2 — 알파 (2~3챕터)

- 홍수 챕터: 환경 자산, 물범람 귀신, 수위 게이지, 미끄럼 바닥
- 쓰나미 챕터: 환경 자산, 파도 거인, 고지대 도달 미션
- 챕터 특화 아이템 (구명조끼, 부유물, 보트, 등산 로프 등)
- DisasterSystem 챕터별 컨피그화 (확률·강제 발동·속도 보너스)
- HealthSystem.drainPerSecond 챕터별 외부화
- 발악점프 판정 (체공 중 체력 0 즉사 X)
- 점프 초기화 정식화 (능력/아이템 종료 시점)
- 비공개 URL 클로즈드 테스트 (지인 10~20명)

### 11.3 Phase 3 — 베타 (4~5챕터 + 메타 프로그레션)

- 지진 챕터: 균열 괴수, 좌우 분기 경로, 낙하물 패턴
- 복합 재난 챕터: 셰이더 합성, 무작위 재난 조합
- 잠금해제 캐릭터 4명 (불꽃, 파도, 기둥, 하늘) + 챕터별 패시브
- 학습 노트 메뉴 (퀴즈 이력 + 정답·해설·출처 모음, 보호자 공유 기능)
- 상점 + 시민 등급 (코인으로 영구 강화 매수)
- 도전과제 시스템
- 점프 캔슬, 땅굴 발악, 능력 발동 후 발악
- Supabase 클라우드 세이브
- 클로즈드 베타 50~100명

### 11.4 Phase 4 — 출시 폴리싱

- 광고 통합 (보상형, 부활형) — AdSense / 네이버 성과형
- 광고 제거 IAP (5,500원), 코인 패키지 IAP
- **스킨 가차 세부 설계 + 구현** (§4.8 본격화)
- 접근성: 진동, 색맹 모드, 사운드 분리(BGM/SFX 음량)
- Supabase 리더보드 (주간/전체)
- PWA 변환 (오프라인 캐시, 홈화면 추가)
